**ls.test.ts tests:**

a) `"ls --json lists files"` — mock chain: tenantTokenResponse() + `{ code: 0, data: { files: [{ name: "file1.docx", type: "docx", token: "tok1" }], has_more: false } }`. Call `ls({ positionals: ["fldcnTest"] }, makeGlobalOpts({ json: true }))`. Assert stdoutJson has `success: true`, `count: 1`, and `files` array.

b) `"ls --type filters by type"` — same mock chain, call with `{ positionals: ["fldcnTest"], type: "docx" }`. Verify the fetch call URL or params contain `type=docx`.

c) `"ls --limit restricts count"` — mock chain with 3 files, call with `{ positionals: ["fldcnTest"], limit: "2" }`. Assert stdoutJson.count is 2 or stdoutJson.files.length is 2.

d) `"ls invalid --limit throws INVALID_ARGS"` — call with `{ positionals: [], limit: "0" }`. Assert rejects with INVALID_ARGS about limit.

e) `"ls invalid --type throws INVALID_ARGS"` — call with `{ positionals: [], type: "invalid_type" }`. Assert rejects with INVALID_ARGS about file type.

f) `"ls empty folder"` — mock with empty files array. json: false. Assert stdout contains "文件夹为空".

g) `"ls human-readable mode"` (per D-04) — mock with one file. json: false. Assert stdout contains the file name and type label (like "[新文档]" for docx).

h) `"ls pagination collects multiple pages"` — mock: tenantTokenResponse() + page 1 `{ code: 0, data: { files: [{ name: "f1", type: "docx", token: "t1" }], has_more: true, next_page_token: "pt1" } }` + page 2 `{ code: 0, data: { files: [{ name: "f2", type: "docx", token: "t2" }], has_more: false } }`. Assert json output has count 2.
