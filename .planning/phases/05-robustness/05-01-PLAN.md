Add configurable retry logic with exponential backoff and jitter to fetchWithAuth and fetchBinaryWithAuth, making the CLI resilient to transient API failures (429 rate limits, 502/503 server errors, timeouts).

Output: src/utils/retry.ts utility, FetchOptions.retry type extension, retry loops in client.ts, comprehensive test coverage.

    The key structural change: wrap the fetch call + status check in a retry loop. The existing error mapping (scope errors, mapApiError) stays OUTSIDE the retry -- only raw HTTP status triggers retry.

    Pseudocode for new fetchWithAuth:
    ```
    1. Resolve base, bearer, url, fetchOpts (unchanged from current)
    2. Determine retryOpts: if options.retry === false -> { maxRetries: 0, ...DEFAULT_RETRY }
       else -> { ...DEFAULT_RETRY, ...options.retry }
    3. Loop: for (let attempt = 0; attempt <= retryOpts.maxRetries; attempt++)
       a. Create fresh AbortController per attempt (CRITICAL: per Pitfall 2)
       b. Set timeout: setTimeout(() => controller.abort(), 30_000)
       c. Try fetch(url, { ...fetchOpts, signal: controller.signal })
       d. On success response: check if isRetryable(res.status) AND attempt < retryOpts.maxRetries
          - If yes: log stderr `feishu-docs: info: API 请求失败（HTTP ${res.status}），第 ${attempt + 1} 次重试...\n`
          - For 429: check parseRetryAfter(res.headers.get("Retry-After")), use that delay or calculateDelay
          - For 502/503: use calculateDelay(attempt, retryOpts.initialDelay, retryOpts.maxDelay)
          - await sleep(delay), continue
          - If retryable but attempt === retryOpts.maxRetries: fall through to normal error handling
       e. On non-retryable response or last attempt: proceed to existing JSON parse + error mapping (unchanged)
       f. On catch: if error.name === "AbortError" AND attempt < retryOpts.maxRetries
          - Log stderr `feishu-docs: info: API 请求失败（超时），第 ${attempt + 1} 次重试...\n`
          - await sleep(calculateDelay(attempt, ...))
          - continue
          - If AbortError and last attempt: throw CliError("API_ERROR", "API 请求超时（30秒）", { retryable: true }) (same as current)
          - If other error: rethrow immediately (no retry)
       g. finally: clearTimeout(timeoutId)
    ```
